The Problem

Manufacturing data is fragmented across dozens of disconnected systems.

SCADA knows machine status but not production targets. MES knows orders but not real-time state. The UNS creates a single source of truth.

Components

Limitless functions. One system.

Each function does one thing well — capture, process, or report. They compose through shared infrastructure.

uns-sim Node.js · MQTT
Simulates 4 CNC machines publishing realistic data every 3 seconds
uns-framework Go · MQTT
Monitors all topics, caches every message to Valkey
uns-cache Node.js · HTTP
Reads cached topics, returns JSON with change detection
uns-log Go · HTTP
Logs snapshot rows to PostgreSQL on data change
uns-state Go · HTTP
Tracks machine state transitions and durations
uns-stoppage Node.js · HTTP
Classifies why machines aren't running
uns-productivity Go · HTTP
Tracks production runs and throughput metrics
uns-input Node.js · HTTP
Manual operator data entry for scrap and notes
uns-kpi Go · HTTP
Computes manufacturing KPIs from all accumulated data
