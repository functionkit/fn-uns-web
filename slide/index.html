The Stack

Three projects. One unified solution.

fn-uns is built on two open foundations — an open standard for structuring manufacturing data, and an open platform for deploying serverless functions.

01

UNS Framework

An open standard that defines how to structure manufacturing data in MQTT. ISA-95 topic hierarchy, versioned namespaces, payload conventions.

Defines the data structure
02

fnkit

A lightweight FaaS platform. Write functions in any language, scaffold with one command, deploy via git push. No cloud vendor required.

Provides the platform
03

fn-uns

Serverless functions implementing a complete UNS data pipeline on fnkit. From machine simulation to KPI computation.

Delivers the functions
How It Works

Structured data. Serverless functions. Git push to deploy.

ISA-95 Topic Hierarchy

All data is organised using the ISA-95 international standard — a structured path from enterprise down to individual data points.

Publish / Subscribe

MQTT decouples producers from consumers. Machines publish to topics, applications subscribe to what they need.

Serverless Functions

Each piece of logic is a small, independent function — Go or Node.js. No monolithic platform, no flow builder.

GitOps Deployment

Every function is version controlled. Deploy with git push. Roll back with git revert.

Components

Limitless functions. Zero coupling.

Each function is a standard Go or Node.js program — its own container, its own repo, its own deploy lifecycle.

Simulates 4 CNC machines publishing realistic status, program, and tool data every 3 seconds

Subscribes to the entire namespace, caches every message with current/previous value tracking

Code-First

Readable. Testable. Reviewable.

No drag-and-drop flow builders. No visual wiring. Just code that any developer can read, review, and extend.

// Read current + previous for every topic
const pipe = rdb.pipeline()

for (const topic of topics) {
  pipe.get(`uns:data:${topic}`)
  pipe.get(`uns:prev:${topic}`)
}

const results = await pipe.exec()

// Did the value change?
const changed = current !== previous

Current value. Previous value. Change detection. One call.

The cache stores two versions of every topic — current and previous. A single pipelined read returns both. The changed flag tells downstream functions whether to act.

Standard Node.js. Standard Redis commands. No proprietary SDK.

Get Started

Running in 5 minutes.

Clone the repo, start the infrastructure, and verify data is flowing. The full UNS pipeline runs locally as a Container.

# Clone and set up the shared network
$ git clone https://github.com/functionkit/fnkit.git && cd fnkit
$ docker network create fnkit-network
$ fnkit cache start
 
# Start the UNS capture pipeline
$ cd uns-framework && docker compose up -d && cd ..
$ cd uns-sim && docker compose up -d && cd ..
$ cd uns-cache && docker compose up -d && cd ..
 
# Verify real-time data is flowing
$ curl http://localhost:8080/uns-cache | jq
